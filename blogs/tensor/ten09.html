GradientTape（）与文本分类

with tf.GradientTape() as tape

根据某个函数的输入变量计算他的导数，with结构会记录下来上下文中所有操作记录，保存在一个tape上，基于这个tape和每次操作产生的导数，用反向微分法计算这些被记录在案的函数导数。

gradientTape默认只监视由tf.variable创建的变量。在针对tf.constant创建的常量时，需要指定watch函数对该常量进行监控。constant并不改变具体的数值（但是可以改变形状、定义值的类型），只是将括号中的数值转变为tf可以接受的张量。


x = tf.constant(3.0)
with tf.GradientTape() as g:
       g.watch(x)
           y = x * x
dy_dx = g.gradient(y, x)
#利用with结构的多层嵌套可以实现高阶求导
x = tf.constant(3.0)
with tf.GradientTape() as g:
      g.watch(x)
      with tf.GradientTape() as gg:
            gg.watch(x)
            y = x * x
dy_dx = gg.gradient(y, x)      
d2y_dx2 = g.gradient(dy_dx, x)

一般情况下，默认gradienttape的资源在一次调用gradient函数后就失效了，再次调用也无法计算。如果需要多次计算梯度，需要设置persistent=True属性。

针对variable变量，一般不需要调用watch函数，gradienttape函数会自动监控数据中的可训练变量。

watch函数是确保指定的tensor被追踪。

# 文本分类代码注释

对于大型网站来说，客户的评论应当是一项颇为关心的数据。从客户的评论数据中，能反映出许多具体而细致的信息。此处使用了TensorFlow提供的电影评论数据集，尝试将数据分为积极（positive）和消极（nagetive）两个对立面。

图片是由一个个像素点组成，每个像素点由具体的灰度值数字表示。然而文本数据并不能通过简单的读取和flatten完成数据处理进入输入层。此处通过一个单词到数值映射的词典函数get_word_index()来实现文本到数值的转换，属实想不到在我看来最难的一步，竟然通过一个函数就解决了。

一个难点解决了，另一个难点接踵而至：神经网络的输入层是固定的，然而评论长度不同转换的数组长度就不同，如何让数组长度保持一致哪？同样，还是一个函数。pad_sequences函数：使数据长度标准化。

现在我们知道了如何将文本变成数值，也知道如何使数组长度一致，那么我们的数据已然可以处理成神经网络可以接受的了。接下来让我们一起来搭建网络，开始训练吧！


# 根据输入的语句判别是正向评论还是负面评论，正面评论为1，负面评论为0
# 预先使用25000条数据进行训练和验证，待参数训练完毕，已经能够根据内容判别出
# 正向还是负向评论，然后去验证测试集评论

import tensorflow as tf
import numpy as np
import matplotlib.pyplot as plt

# 导入IMDB数据，从此数据中读取到训练集和测试集
imdb=tf.keras.datasets.imdb
(train_data,train_labels),(test_data,test_labels)=imdb.load_data(num_words=10000)
# 打印发现，训练集和测试集都有25000，对应标签25000，都是以列表的形式存在的
# num_words表示的是一个单词库，只取常用的前10000个单词，所有超过10000的会根据要求被设置为0、1、2、3

# 一个映射单词到整数索引的词典
word_index = imdb.get_word_index()

# 对于索引中的内容将文本和索引对应文件放到word_index中，使value值从4开始
word_index = {k:(v+3) for k,v in word_index.items()}

#sorted_word_index = sorted(word_index.items(), key = lambda d:d[1])
# sorted函数：排序函数，第一个参数表示需要排序的内容，第二个参数表述根据谁进行排序。如果待排序内容是元组或者列表，则不需要后边的部分，如果是字典类型，
# 冒号前后可以是任意字母，表示元素，冒号后如果参数是0，表示以字典的第一个参数排序，1 以第二个参数排序
#print(sorted_word_index[0:100]) 打印出来之后发现，imdb是从4开始排序的，也就解释了前边为什么要加三
# 在value[0-3]的位置存放几个特殊字符
word_index["<PAD>"] = 0
word_index["<START>"] = 1
word_index["<UNK>"] = 2  # unknown
word_index["<UNUSED>"] = 3
# 将word_index 中的key值和value值互换位置
reverse_word_index = dict([(value, key) for (key, value) in word_index.items()])

def decode_review(text):
    # 定义一个解码函数
    return ' '.join([reverse_word_index.get(i, '?') for i in text])
    # 以空格为分割，join函数用于连接字符串

# keras只能接受序列相同的输入，因此要对原始数据进行填充，使其变为序列相同的数据
train_data = tf.keras.preprocessing.sequence.pad_sequences(train_data,
                                                        value=word_index["<PAD>"],  #浮点数，此值将在填充时代替默认的填充值0
                                                        padding='post',   #pre在起始补0，post在末尾补0
                                                        maxlen=256)   #序列最大长度，过短填充0，过长截断

test_data = tf.keras.preprocessing.sequence.pad_sequences(test_data,
                                                       value=word_index["<PAD>"],
                                                       padding='post',
                                                       maxlen=256)

# 构建网络模型
vocab_size=10000   #输入形状是用于电影评论的词汇数目
model = tf.keras.Sequential([
    tf.keras.layers.Embedding(vocab_size,16),
    #前一个表示输入数据，后一个参数表示全连接嵌入维度(并不是神经元个数)
    #嵌入层（Embedding Layer）是使用在模型第一层的一个网络层，其目的是将所有索引标号映射到致密的低维向量中
    # 既可以升维，也可以降维，降维为了获取某些表观特征，升维为了获取某些细致特征.
    # 每一句话都通过一个二维数组来表示，是多个文本，则可以变为三维
    # embedding层只能用在第一层
    tf.keras.layers.GlobalAveragePooling1D(),
    tf.keras.layers.Dense(16, activation='relu'),
    tf.keras.layers.Dense(1,activation='sigmoid')
    ])

# 优化
model.compile(optimizer='adam',
              loss='binary_crossentropy',
              # 二分类的交叉熵函数
              metrics=['accuracy'])

# 创建验证集
# 目的：从训练集分割出来一部分数据作为验证集。
# 在训练过程中，只使用训练集的数据进行训练，15000个数据用来训练，10000个数据用来验证
x_val=train_data[:10000]
partial_x_train = train_data[15000:]
y_val=train_labels[:10000]
partial_y_train = train_labels[15000:]

# 训练
history=model.fit(partial_x_train, partial_y_train, batch_size=512, epochs=40, validation_data=(x_val, y_val), verbose=1)
# verbose是日志显示，有三个参数可选择，分别为0,1和2。当verbose=0时，简单说就是不输出日志信息 ，进度条、loss、acc这些都不输出。当verbose=1时，带进度条的输出日志信息
# 当verbose=2时，为每个epoch输出一行记录，和1的区别就是没有进度条

# 分析结果
results=model.evaluate(test_data,test_labels,verbose=2)
print(results)

# 可视化
history_dict=history.history

acc=history_dict['accuracy']
val_acc=history_dict['val_accuracy']
loss=history_dict['loss']
val_loss=history_dict['val_loss']

epochs=range(1,len(acc)+1)

# plt.plot(x, y, format_string, **kwargs)
# x：x轴数据，列表或数组，可选
# y：y轴数据，列表或数组
# format_string：控制曲线的格式字符串，可选，由颜色字符、风格字符和标记字符组成
# **kwargs,第二组或更多，(x,y,format_string)
plt.plot(epochs,loss,'bo',label='Training loss')
plt.plot(epochs,val_loss,'b',label='Training val_loss')
plt.title('Training and validation loss')
plt.xlabel('Epochs')
plt.ylabel('Loss')
plt.legend()
# 设置图例，又三个参数，可以设置展示的图是否有边框，背景，和图的位置，字体大小等
plt.show()


plt.plot(epochs, acc, 'bo', label='Training acc')
plt.plot(epochs, val_acc, 'b', label='Validation acc')
plt.title('Training and validation accuracy')
plt.xlabel('Epochs')
plt.ylabel('Accuracy')
plt.legend()

plt.show()
